## 摘要

arXiv Reader 是本地已有的一个论文自动推送工具。

它的目标是每天获取指定领域的 arXiv 最新论文，用模型生成中文标题和摘要，再通过邮件发送出来。对我来说，它不是为了替代精读，而是为了降低“每天重新打开论文信息流”的成本。

## 它解决什么问题

论文信息流最大的问题不是没有内容，而是太多。

如果每天都手动去看，很容易被数量压垮；如果完全不看，又会错过一些和当前方向相关的线索。arXiv Reader 更适合作为第一层筛选：先把候选内容收进来，翻译成中文摘要，再决定哪些值得继续看。

## 当前能力

从本地 README 看，它的设计包括几块：

- 按 arXiv 分类抓取论文；
- 使用模型翻译标题和摘要；
- 通过 SMTP 邮件发送 HTML 推荐；
- 支持定时任务；
- 提供 Web API 和网页查看每日数据；
- 支持关键词关注和忽略规则。

这些功能都围绕一个目标：把论文发现变成稳定流程，而不是临时搜索。

## 我的使用期待

我希望它之后能成为研究信息流的一部分。

每天不需要读完所有论文，只需要知道今天有哪些方向值得注意。真正的价值不在“推送了多少篇”，而在于能不能持续筛出少量值得进一步看的内容。
